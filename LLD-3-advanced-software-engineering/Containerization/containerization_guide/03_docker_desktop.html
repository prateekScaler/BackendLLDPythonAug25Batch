Docker Desktop vs Engine

Why Desktop is banned and what to use instead

Wait, There Are Two Dockers?

This confuses a lot of people. "Docker" actually refers to several different things. The two main ones:

Docker Engine

  • Open source (Apache 2.0 license)
  • Free forever, for everyone
  • The actual container runtime
  • Linux only (native)
  • CLI-based

Docker Desktop

  • Commercial/proprietary product
  • Paid subscription for large companies*
  • GUI app for Mac/Windows
  • Includes VM + Engine + extras
  • Nice UI and integrations
Key distinction: Docker Engine (the runtime) is and will always be free and open source. Docker Desktop (the Mac/Windows app) has commercial licensing restrictions.

Docker Desktop on 1000 laptops

In August 2021, Docker Inc. changed their licensing. The new terms:

Docker Desktop requires a paid subscription if:
  • Your company has 250+ employees, OR
  • Your company has more than $10 million in annual revenue
Cost: $5-21 per user per month. For 1000 developers, that's $60K-$252K per year!

Most large companies decided it's easier to ban Docker Desktop entirely rather than manage compliance (who's allowed to use it, auditing installations, etc.). Hence the blanket bans.

Important: Banning Docker Desktop â‰  Banning Docker. You can still use containers! You just need an alternative way to run the Linux VM on Mac/Windows.

Why Mac/Windows Need a VM Anyway

"Wait, if Docker Engine runs on Linux, how does it work on my Mac?"

Good question! Docker uses Linux kernel features (namespaces, cgroups). macOS has the Darwin kernel, Windows has NT. Neither has these features. So you need a Linux VM running in the background.
