The Four Golden Signals

Google's SRE book recommends monitoring these four metrics for any service:

â±ï¸

1. Latency

How long requests take. Track p50, p90, p99 percentiles, not just average!

Example: "p99 latency is 500ms" means 99% of requests are faster than 500ms

ğŸ“ˆ

2. Traffic

How much demand is on your system. Requests per second, concurrent users, etc.

Example: "1,000 req/sec to the API gateway"

âŒ

3. Errors

Rate of failed requests. Both explicit (500s) and implicit (wrong data, timeouts).

Example: "0.1% error rate" or "50 errors in last 5 minutes"

4. Saturation

How "full" your service is. CPU, memory, disk, connections, queue depth.

Example: "Database connection pool at 80% capacity"

What is Prometheus?

Prometheus is the industry-standard open-source metrics system. It collects metrics from your services, stores them as time-series data, and lets you query/alert on them. Used by: Kubernetes, GitLab, DigitalOcean, and most modern infrastructure.

Prometheus Metric Examples

# Counter: Total HTTP requests
http_requests_total{method="GET", path="/api/users", status="200"} 1234567

# Gauge: Current active connections
active_connections{service="api-gateway"} 42

# Histogram: Request duration buckets
http_request_duration_seconds_bucket{le="0.1"} 10000  # 10k requests < 100ms
http_request_duration_seconds_bucket{le="0.5"} 15000  # 15k requests < 500ms
http_request_duration_seconds_bucket{le="1.0"} 16000  # 16k requests < 1s
http_request_duration_seconds_bucket{le="+Inf"} 16500 # All requests

Database Metrics

  • Query latency: How long queries take (p50, p95, p99)
  • Connection pool: Active, idle, waiting connections
  • Replication lag: How far behind replicas are
  • Lock waits: Time spent waiting for locks
  • Cache hit ratio: Buffer pool efficiency

API Metrics: What and Why

Click each metric to reveal what it tells you and when to alert:

Request Rate

What it tells you: Traffic volume, usage patterns

Alert when: Sudden drop (outage?) or spike (attack?)

Error Rate

What it tells you: Service health, bugs in code

Alert when: > 1% of requests failing

p50 Latency

What it tells you: Typical user experience (median)

Alert when: > 200ms for typical APIs

p99 Latency

What it tells you: Worst-case user experience

Alert when: > 1000ms (varies by service)

Availability

What it tells you: Uptime percentage

Alert when: < 99.9% (depending on SLA)

Throughput

What it tells you: Data processed per second

Alert when: Approaching capacity limits
